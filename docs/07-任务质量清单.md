# 任务质量清单

本清单用于判断一个任务是否适合进入团队内部 AI Agent 测试集。它适用于内置 `tasks/<task-id>/`，也适用于私有 `evalsets/<set>/tasks/<task-id>/`。

## 结构完整

- `task.yaml` 存在，`task_id` 等于目录名。
- `work/` 存在，且只包含 Agent 可见材料。
- `hidden/judge.rules.yaml` 存在。
- `samples/attempt-pass.json`、`samples/attempt-fail.json`、`samples/replay.yaml` 都存在。
- `bin/agent-eval validate --task <task-dir>` 通过。
- scripted 回放能跑出 fail -> pass 闭环。

## 题面质量

- `agent_brief` 只描述任务目标、材料位置、提交格式，不夹带答案线索。
- `visible_context` 覆盖所有必要材料。
- Agent 不需要读 `hidden/` 才能完成任务。
- 任务有真实业务含义，不只是为了覆盖某个 check 类型。
- 任务有区分度：朴素 Agent 或粗糙策略会犯错，合格 Agent 能通过。

## 判分质量

- 优先使用确定性 check：`json_schema`、`jsonpath_equals`、`list_coverage`、`command`、`tool_call_required`、`world_state` 等。
- hidden 期望值放在 `hidden/expected/`，通过 `expected_from` 引用。
- `feedback_fail` 只给方向，不泄露 expected 值、私有规则细节或 mock 响应。
- blocking 只用于关键契约或核心安全项，不用于低权重主观项。
- `llm_rubric` 只能低权重、非 blocking，不能作为一票否决。
- 维度权重总和等于 `scoring.max_score`。

## 样例质量

- pass 样例代表满分路径，不靠绕过题面或硬编码框架行为通过。
- fail 样例犯真实 Agent 可能犯的错误，例如漏读材料、误报、漏报、工具未调用、引用无效来源。
- fail 样例不应只是 JSON 格式错误；格式错误测的是接入契约，不是业务能力。
- replay 编排应体现第一轮失败、收到受控反馈后第二轮或后续轮次修正通过。
- 样例中不得复制 hidden expected、judge 私有规则或 mock 响应。

## 安全与防泄露

- `hidden/expected` 不出现在 `work/`、`samples/`、`agent_brief`、公开文档、PR 描述或反馈文案。
- `hidden/tools/*.responses.yaml` 不出现在 Agent 可见材料。
- canary token 不出现在公开材料。
- 真实凭证只通过环境变量引用，例如 `${ENV:API_TOKEN}`。
- 任务作者不能把 `judge/*.judge.json` 的 `private_notes` 原文回灌到 Agent 提示里。

## 分层建议

| 分层 | 进入标准 |
| --- | --- |
| `smoke` | 快、稳定、覆盖核心能力；每次改 Agent 必跑 |
| `regression` | 有业务代表性和区分度；合并前或发版前跑 |
| `domain` | 专项能力或业务线任务；用于诊断和选型 |
| `security` | 越权、泄露、工具权限、安全边界类任务；失败需人工确认 |

## 审查结论模板

```text
结论：ready:regression
建议分层：regression

主要依据：
- validate 通过
- scripted 回放 fail -> pass 通过
- fail 样例覆盖真实高概率错误
- feedback_fail 未泄露 expected

进入门禁前建议：
- 增加一个工具未调用的 fail 样例
- 把 pass_score 从 70 调到 80
```

## 最小验证命令

```bash
bin/agent-eval validate --task <task-dir>
bin/agent-eval run --task <task-dir> --agent scripted \
    --script <task-dir>/samples/replay.yaml
```

改内置 `tasks/` 后，还要运行：

```bash
bin/agent-eval suite --tasks-root tasks --fail-on-not-passed
```
