# 缺陷报告

线上价格聚合服务在部分订单上抛出异常，已定位到 `src/PriceCalculator.java` 的 `total` 方法。

## 已知现象

- 传入普通价格列表（如 `[100, 250, 30]`）时抛出异常，未能返回合计 `380`；
- 传入空列表时期望返回 `0`；
- 上游偶尔会传入 `null`（历史原因短期无法修改上游），期望按 `0` 处理而不是抛 NPE。

## 约束

- 保持类名 `PriceCalculator` 与方法签名 `public int total(List<Integer> prices)` 不变；
- 不得破坏同类中已有的其它方法（如 `maxPrice`）的现有正确行为（回归保护）；
- 不引入第三方依赖。
